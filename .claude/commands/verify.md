Проведи верификацию реализованного кода на соответствие спецификациям из `docs/`.

## Алгоритм

1. **Определи реализованные слои** — посмотри, какие crate'ы есть в `crates/`, какие коммиты были сделаны
2. **Для каждого реализованного слоя** прочитай соответствующую спеку:
   - `crates/kernel/` → `docs/layer1_spec.md`
   - `crates/db/` → `docs/layer2_spec.md`
   - `crates/event_bus/` → `docs/layer3a_spec.md`
   - `crates/auth/` → `docs/layer4a_spec.md`
   - `crates/runtime/` → `docs/layer5a_spec.md`
   - Workspace/Cargo.toml → `docs/layer0_spec.md`
3. **Сверь по чеклисту** для каждого слоя:
   - Все ли структуры/трейты/enums из спеки реализованы?
   - Совпадают ли сигнатуры (имена полей, типы, методы)?
   - Реализованы ли все тесты, перечисленные в спеке?
   - Соблюдены ли зависимости между crate'ами (что от чего зависит)?
   - Нет ли лишнего, чего нет в спеке (over-engineering)?
4. **Выведи отчёт** в формате:

```
## Верификация: Layer N — <название>
Спека: docs/layerN_spec.md

### Соответствует
- [x] item 1
- [x] item 2

### Расхождения
- [ ] что не реализовано или отличается — и что нужно сделать

### Лишнее (нет в спеке)
- item — оценка: нужно/можно удалить
```

5. **Итоговая сводка** — общий статус по всем слоям, критичные расхождения.

Если аргумент указан (например `/verify layer2`), проверь только указанный слой.
